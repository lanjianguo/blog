浅谈CSS盒模型

发表于 2018-11-12 | 分类于 css |

简述

首先先解释一下，什么是css的盒模型。先看一下百度百科对它对定义：先说说我们在网页：内容(content)、内边距(padding)、边框(border)、外边距(margin)， CSS盒子模式都具备这些属性。
这些属性我们可以把它转移到我们日常生活中的盒子（箱子）上来理解，日常生活中所见的盒子也就是能装东西的一种箱子，也具有这些属性，所以叫它盒子模式。

其实也就是说，因为网页里对要素，具备了像我们常见的盒子一样的一些属性，所以我们就将网页中的要素，抽象成盒子模型。

因为浏览器厂商对于标准的不统一，所以就造成了市面上两种盒子模型的存在，一种是W3C的标准盒模型，一种IE的盒模型。

解析

虽然是有两种盒模型的存在，但是对于盒模型的属性，都是一样的，二者的区别在于宽度盒高度定义的不同，后面会说到。
盒子模型的要素包括以下内容：

  • content：盒子的核心，内容区域
  • padding：内边距区域padding area延伸到包围padding的边框
  • border：由border和4条border edge组成。若border宽度设置为0，则border edge与padding edage重叠
  • margin box：由margin和4条margin edge组成。若margin宽度设置为0，则margin edge与border edage重叠。

W3C的标准盒模型

在标准的盒子模型中，width指content部分的宽度

IE的盒模型

在IE盒子模型中，width表示content+padding+border这三个部分的宽度

转换

如果想要切换盒模型则可以使用css3的box-sizing属性

  box-sizing: content-box 是W3C盒子模型
  box-sizing: border-box 是IE盒子模型

box-sizing的默认属性是content-box
